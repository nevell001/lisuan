# PDF 导出时间格式优化文档

## 优化概述

本次优化改进了 PDF 导出功能中的时间格式显示，使日期时间信息（如 `2026-02-23 22:13:43`）能够自动分成两行显示，提升可读性。

## 优化内容

### 1. 时间格式自动分行

**修改文件**: `src/main/java/com/cashier/util/ExportUtil.java`

**新增方法**:
- `isDateTimeFormat(String text)` - 检测文本是否为日期时间格式
- `splitDateTimeToLines(String text)` - 将日期时间格式分割成两行

**优化方法**:
- `splitTextIntoLines(String text, PDFont font, float fontSize, float maxWidth)` - 支持日期时间格式自动分行

**功能说明**:
- 当检测到日期时间格式（匹配正则表达式 `\d{4}[-/]\d{1,2}[-/]\d{1,2}\s+\d{1,2}:\d{2}(:\d{2})?`）时
- 自动将文本分成两行显示：
  - 第一行：日期部分（如 `2026-02-23`）
  - 第二行：时间部分（如 `22:13:43`）
- 只有当两行都能适应列宽时才使用分行显示，否则按普通文本处理

### 2. 时间列宽度优化

**修改方法**: `calculateColumnWidths(List<String> headers, List<String[]> data, PDFont font, float fontSize, float totalWidth)`

**优化内容**:
- 新增时间列标记（`isDateTimeColumn` 数组）
- 对时间列的宽度计算进行特殊处理：
  - 如果是时间列且包含日期时间格式，则按两行计算宽度
  - 取日期和时间两部分中的最大宽度作为列宽
  - 避免时间列被计算得比实际需要的要大
- 时间列最小宽度从 150px 调整为 140px（因为按两行计算后不需要那么宽）

## 支持的时间格式

支持以下格式（使用正则表达式匹配）：
- `2026-02-23 22:13:43`（标准格式）
- `2026/02/23 22:13`（斜杠分隔，无秒）
- `2026-2-23 2:13:43`（单数字月份/小时）
- 以及类似的日期时间组合格式

## 技术细节

### 时间格式检测正则表达式
```java
"\\d{4}[-/]\\d{1,2}[-/]\\d{1,2}\\s+\\d{1,2}:\\d{2}(:\\d{2})?"
```

**说明**:
- `\d{4}` - 4位年份
- `[-/]` - 横杠或斜杠分隔符
- `\d{1,2}` - 1-2位月份
- `\d{1,2}` - 1-2位日期
- `\s+` - 一个或多个空格
- `\d{1,2}` - 1-2位小时
- `:\d{2}` - 分钟（必须有）
- `(:\d{2})?` - 秒数（可选）

### 列宽计算优化

对于时间列，计算宽度时：
```java
if (isDateTimeColumn[i] && isDateTimeFormat(cell)) {
    String[] parts = splitDateTimeToLines(cell);  // 分割成日期和时间
    float maxPartWidth = 0;
    for (String part : parts) {
        float partWidth = font.getStringWidth(part) / 1000 * fontSize;
        maxPartWidth = Math.max(maxPartWidth, partWidth);  // 取最大宽度
    }
    maxWidths[i] = Math.max(maxWidths[i], maxPartWidth);
}
```

这样计算的是日期和时间两行中较宽的那一行的宽度，而不是整行文本的宽度。
