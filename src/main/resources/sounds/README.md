# 扫描音效文件说明

本目录用于存放 POS 扫描功能的音效文件。

## 需要的音效文件

1. **scan_success.mp3** - 扫描成功音效
   - 当商品成功添加到购物车时播放
   - 建议：短促的"叮"声或"滴"声
   - 时长：建议 0.2-0.5 秒

2. **scan_error.mp3** - 扫描错误音效
   - 当商品库存不足或其他错误时播放
   - 建议：低沉的"嗡"声或"错误"提示音
   - 时长：建议 0.3-0.6 秒

3. **scan_not_found.mp3** - 扫描未找到音效
   - 当扫描的商品不存在时播放
   - 建议：较长的提示音，如"嘟-嘟"
   - 时长：建议 0.5-1.0 秒

## 音效文件要求

- **格式**：MP3 或 WAV
- **采样率**：44.1kHz 或 48kHz
- **比特率**：128kbps 或更高
- **时长**：0.2-1.0 秒（保持短促）
- **音量**：适中，不要过大或过小

## 如何获取音效

### 在线音效库

- Freesound.org (免费音效库)
- Zapsplat.com (免费音效库)
- SoundBible.com (免费音效库)

搜索关键词：
- "beep"
- "ding"
- "success"
- "error"
- "notification"
- "scanner"

### 自己制作

可以使用 Audacity 等免费音频编辑软件制作简单的音效：
1. 录制或生成正弦波
2. 调整频率和时长
3. 添加淡入淡出效果
4. 导出为 MP3 格式

## 示例音效

如果暂时没有音效文件，系统会静默运行，不会影响基本功能。
建议尽快添加音效以提升用户体验。

## 音效文件位置

确保音效文件放置在正确的位置：
```
src/main/resources/sounds/
├── scan_success.mp3
├── scan_error.mp3
└── scan_not_found.mp3
```

## 测试音效

启动应用后，尝试扫描商品条码：
- 扫描成功商品：播放 scan_success.mp3
- 扫描库存不足商品：播放 scan_error.mp3
- 扫描不存在的商品：播放 scan_not_found.mp3

## 故障排除

如果音效无法播放：
1. 检查文件名是否正确
2. 检查文件格式是否为 MP3 或 WAV
3. 检查文件是否损坏
4. 查看控制台日志获取详细错误信息
